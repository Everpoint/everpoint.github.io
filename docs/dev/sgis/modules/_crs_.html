Projection

Projection: function
  • Defined in Crs.ts:9

A function that receives a pair of coordinates and converts them to the coordinates in another coordinate system. This type shall be used for high performance mass converting of coordinates and for declaring custom coordinate systems. In other cases .projectTo(crs) methods are preferred.

Const geo

geo: Crs = new Crs({wkid: 4326,authority: 'EPSG',wkt: EPSG4326_WKT})
  • Defined in Crs.ts:169

Geographic coordinate system on WGS84 datum with latitude being first coordinate, and longitude as second coordinate.
NOTE: This coordinate system is registered by EPSG as EPSG:4326, but a lot of GIS software implements 4326 as longitude/latitude. If you are used to that coordinate order use wgs84 crs instead of this one.

Coordinate system used by many gis web applications. The coordinates are given in meters, as projected by Mercator projection using WGS84 datum assuming that the Earth is a sphere. Has high distortions near the poles.

Const wgs84

wgs84: Crs = new Crs({wkid: 84,authority: 'OCG',wkt: OCG84_WKT})
  • Defined in Crs.ts:148

Geographic coordinate system on WGS84 datum, which has longitude set as first coordinate, and latitude as second coordinate.
NOTE: A lot of GIS software incorrectly implement EPSG:4326 as this longitude/latitude coordinate system. If you are used to using such systems as OpenLayers or PostGIS, you want to use this crs instead of 4326 in those system.
